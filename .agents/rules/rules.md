# 1. 项目定位与核心技术栈
- **项目性质**：本项目是一个纯前端的静态网页工具集，主要用于教学管理、班级互动等场景。
- **技术栈限制**：严格使用纯 HTML、CSS 和原生 JavaScript（Vanilla JS）。
- **绝对禁区**：禁止使用 Node.js、React、Vue 等复杂框架；禁止引入 Webpack、Vite 等构建打包工具；禁止写任何需要服务器后端（如 Python/PHP）才能运行的代码。
- **部署环境意识**：项目托管在 GitHub，由 Cloudflare Pages 提供静态代理。所有的内部文件引用（图片、音效、跳转链接）**必须使用相对路径**（例如 `./style.css` 或 `../images/bg.png`），避免在 Cloudflare 上出现 404 找不到文件的情况。

# 2. Agent (Antigravity) 专属行为规范
作为在 Antigravity 中运行的 AI，你的首要任务是直接修改文件，而不是在聊天框里废话：
- **拒绝输出完整代码**：在聊天框中，绝对不要输出包含几百行的完整 HTML/JS 代码。请直接利用 Antigravity 的文件编辑能力，对目标文件进行**精准的局部修改（Diff 替换）**。
- **说人话，先解释后动手**：因为用户是非专业程序员，在修改代码前，请用简短、通俗的中文解释一下“问题出在哪里”以及“你打算怎么改”。不要使用晦涩的计算机底层术语。
- **写详细的中文注释**：在你新增或修改的 JavaScript 逻辑和关键 CSS 样式旁，必须加上通俗易懂的中文注释，解释这段代码的作用，方便用户日后自行调整参数。

# 3. 业务数据与语言规范
- **原样保留业务数据**：用户代码中出现的任何中文文本、学生名单、班级信息、特定选项、测试数据等，都是实际业务需要。**绝对禁止**擅自将其打码、删除、或替换为 `[Your Name]`、`John Doe`、`Test1` 等无意义的英文占位符。
- **编码与语言**：强制所有新建文件使用 UTF-8 编码。网页的语言设置应为 `<html lang="zh-CN">`。任何你生成的面向用户的弹窗提示（alert）、按钮文字，必须是符合中国大陆习惯的简体中文。
- **第三方库的使用**：如果必须要实现某个复杂功能（如生成二维码、导出 Excel、复杂图表），允许通过 CDN 引入成熟的第三方轻量级 JS 库（如 SheetJS、SweetAlert2），但必须在引入时说明用途。
